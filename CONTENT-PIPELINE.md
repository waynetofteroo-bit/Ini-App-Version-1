# ini — Content Pipeline

Read before working on `/api/ingest` or any content seeding.

---

## Overview

Questions and knowledge graph nodes are generated by a four-agent pipeline, not written by hand. The pipeline runs on-demand via `POST /api/ingest` (admin only).

**Important:** Generate and seed content unit-by-unit to avoid Claude API token limits. Do not attempt to generate all 34 topics in a single call.

---

## Four-agent pipeline

Each agent is a separate `callClaudeJSON()` call. Output of each feeds as input to the next.

```
Spec content (text)
  └── Agent 1: Extractor   → raw topic list + learning outcomes
        └── Agent 2: Classifier → Bloom-tagged outcomes + taxonomy
              └── Agent 3: Auditor → validated questions
                    └── Agent 4: Graph Builder → knowledge_graph_nodes + edges
```

---

## Agent 1 — Extractor

**Input:** Raw WJEC spec text for a unit (passed as `userMessage`)

**System prompt:**
```
You are a curriculum analyst. Extract all learning outcomes from the provided WJEC specification text.

For each topic in the text, return:
- topic_label: short name for the topic
- learning_outcomes: array of outcome strings, verbatim from the spec
- command_words: array of command words used (e.g. describe, explain, calculate, evaluate)

Return JSON only: { topics: [{ topic_label, learning_outcomes, command_words }] }
```

---

## Agent 2 — Classifier

**Input:** Agent 1 JSON output

**System prompt:**
```
You are a Bloom's taxonomy specialist for UK physics education.

For each learning outcome provided, assign:
- bloom_level: integer 1–5
  - 1 (Remember): recall, state, name, list
  - 2 (Understand): describe, explain, summarise
  - 3 (Apply): calculate, use, apply, determine
  - 4 (Analyse): analyse, compare, explain why, interpret
  - 5 (Evaluate/Create): evaluate, justify, design, assess, discuss

Also generate:
- 3–5 MCQ questions per topic covering Bloom L1–L3
- 1–2 extended response question stems per topic covering Bloom L4–L5

Return JSON only matching this schema:
{
  "topics": [{
    "topic_label": string,
    "concept_uri": string,   // format: "wjec:physics:{unit_slug}:{topic_slug}"
    "bloom_ceiling": number,
    "outcomes": [{
      "text": string,
      "bloom_level": number
    }],
    "mcq_questions": [{
      "stem": string,
      "options": [{"idx": 0, "text": string}, ...],  // always 4 options
      "correct_idx": number,
      "bloom_level": number,
      "explanation": string
    }],
    "extended_questions": [{
      "stem": string,
      "bloom_level": number,
      "markscheme_points": string[],
      "indicative_content": string,
      "rubric_bands": {
        "Full":    {"descriptor": string, "marks": number},
        "Good":    {"descriptor": string, "marks": number},
        "Partial": {"descriptor": string, "marks": number},
        "Minimal": {"descriptor": string, "marks": number}
      }
    }]
  }]
}
```

---

## Agent 3 — Auditor

**Input:** Agent 2 JSON output

**System prompt:**
```
You are a question quality auditor for WJEC GCSE Physics.

For each MCQ question, check:
1. Stem clarity: is the question unambiguous?
2. Distractor validity: are wrong options plausible but clearly incorrect?
3. Bloom alignment: does the question actually test the stated Bloom level?
4. Correct answer: is the correct_idx definitively correct?

For each extended response question, check:
1. Stem is open enough to allow Full band responses
2. Markscheme points are distinct and non-overlapping
3. Rubric bands are clearly differentiated

Return the same JSON structure with:
- flagged: true/false per question
- flag_reason: string (if flagged)
- Corrected questions where you can fix issues automatically
- Remove questions you cannot fix (too ambiguous or factually incorrect)
```

---

## Agent 4 — Graph Builder

**Input:** Agent 3 audited JSON output + existing `knowledge_graph_nodes` for the course (to detect prerequisites)

**System prompt:**
```
You are building a knowledge graph for WJEC GCSE Physics.

For each topic provided, create:
1. A knowledge_graph_node with concept_uri, label, bloom_ceiling, topic_tier (core/higher)
2. knowledge_graph_edges for prerequisite relationships:
   - A topic is a prerequisite of another if understanding it is required first
   - Use relation: "prerequisite"
3. knowledge_graph_edges for bridge relationships:
   - Topics from different units that share concepts
   - Use relation: "bridge"

Return JSON only:
{
  "nodes": [{
    "concept_uri": string,
    "label": string,
    "bloom_ceiling": number,
    "topic_tier": "core" | "higher",
    "gap_flags": []
  }],
  "edges": [{
    "from_concept_uri": string,
    "to_concept_uri": string,
    "relation": "prerequisite" | "bridge"
  }]
}
```

## `/api/ingest` route logic

```typescript
// POST body: { unit_id: string, spec_content: string }
// Headers: x-admin-secret: process.env.ADMIN_SECRET

// 1. Validate admin secret
// 2. Fetch unit record from DB to confirm it exists
// 3. Run Agent 1 (Extractor)
// 4. Run Agent 2 (Classifier) — pass Agent 1 output as userMessage
// 5. Run Agent 3 (Auditor) — pass Agent 2 output as userMessage
// 6. Run Agent 4 (Graph Builder) — pass Agent 3 output as userMessage
// 7. Insert knowledge_graph_nodes (with unit_id FK)
// 8. Insert knowledge_graph_edges (look up node IDs by concept_uri)
// 9. Insert questions (look up concept_id by concept_uri)
//    - For L4-L5: build marking_prompt JSONB from extended_questions data
// 10. Return summary: { nodes_created, questions_created, edges_created }
```

## Seeding strategy

### Initial seed (run once after schema migration)

```sql
-- 1. Insert course
insert into courses (course_code, course_name, exam_board, level, subject)
values ('WJEC-GCSE-PHY-DA', 'WJEC GCSE Physics Double Award', 'WJEC', 'GCSE', 'Physics')
returning id;

-- 2. Insert units (replace {course_id} with returned id)
insert into units (course_id, unit_code, unit_name, unit_order, exam_board) values
  ('{course_id}', 'WJEC-DA-U1', 'Electricity',        1, 'WJEC'),
  ('{course_id}', 'WJEC-DA-U2', 'Forces and Motion',  2, 'WJEC'),
  ('{course_id}', 'WJEC-DA-U3', 'Waves',              3, 'WJEC'),
  ('{course_id}', 'WJEC-DA-U4', 'Energy',             4, 'WJEC'),
  ('{course_id}', 'WJEC-DA-U5', 'The Universe',       5, 'WJEC'),
  ('{course_id}', 'WJEC-DA-U6', 'Particles',          6, 'WJEC');
```

### Question generation (run per unit after seed)

Call `POST /api/ingest` once per unit with the spec text for that unit:

```bash
curl -X POST https://your-app.vercel.app/api/ingest \
  -H "x-admin-secret: $ADMIN_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "unit_id": "uuid-of-wjec-da-u1",
    "spec_content": "Unit 1: Electricity\n1.1 Electric charge...[full spec text]"
  }'
```

Run this for each of the 6 units. Do not batch — one unit per call to stay within token limits.

## concept_uri format

All concept URIs follow the pattern: `{board}:{subject}:{unit_slug}:{topic_slug}`

Examples:
```
wjec:physics:electricity:ohms-law
wjec:physics:electricity:series-parallel-circuits
wjec:physics:forces-motion:newtons-second-law
wjec:physics:waves:transverse-longitudinal
```

Slugs are lowercase, hyphen-separated, no special characters.

---

## Adding a new exam board or subject

1. Insert a new `courses` row with the new `course_code`, `exam_board`, `level`, `subject`
2. Insert `units` rows for that course (each `knowledge_graph_nodes` graph is scoped per board via `exam_board` column)
3. Run `/api/ingest` per unit with spec content for the new board
4. The graph for WJEC and AQA are separate — no cross-contamination
